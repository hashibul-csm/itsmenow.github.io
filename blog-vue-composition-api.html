Vue.js 3 Composition API: Best Practices

November 28, 2024 • 7 min read • Hashibul Hasan

Vue.js 3 introduced the Composition API as a more flexible and powerful way to organize component logic. While the Options API is still perfectly valid, the Composition API offers better code organization, reusability, and TypeScript support. In this guide, we'll explore best practices for using the Composition API effectively.

Understanding the Composition API

The Composition API allows you to organize component logic using imported functions instead of defining options in the component object. This approach makes it easier to reuse logic across components and provides better TypeScript support.

Basic Setup

<template>
  <div>
    <h1>{{ title }}</h1>
    <p>Count: {{ count }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

const count = ref(0)
const title = ref('Composition API Demo')

const increment = () => {
  count.value++
}

const doubleCount = computed(() => count.value * 2)
</script>

Best Practices

1. Use <script setup> Syntax

The <script setup> syntax is the recommended way to use the Composition API. It provides more concise code and better performance.

2. Organize Logic with Composables

Extract reusable logic into composables. This promotes code reusability and better organization. 

// useCounter.js
import { ref, computed } from 'vue'

export function useCounter(initialValue = 0) {
  const count = ref(initialValue)

  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => count.value = initialValue

  const double = computed(() => count.value * 2)

  return {
    count,
    increment,
    decrement,
    reset,
    double
  }
}

3. Proper Lifecycle Management

Use the appropriate lifecycle hooks and clean up resources properly. 

import { onMounted, onUnmounted } from 'vue'

export function useMousePosition() {
  const x = ref(0)
  const y = ref(0)

  const updatePosition = (event) => {
    x.value = event.clientX
    y.value = event.clientY
  }

  onMounted(() => {
    window.addEventListener('mousemove', updatePosition)
  })

  onUnmounted(() => {
    window.removeEventListener('mousemove', updatePosition)
  })

  return { x, y }
}

4. Reactive Data Patterns

Use reactive() for complex objects and ref() for primitive values. 

import { reactive, ref, computed } from 'vue'

// For objects
const user = reactive({
  name: 'John',
  age: 30
})

// For primitives
const count = ref(0)

// For computed properties
const fullName = computed(() => `${user.name} (${user.age})`)

5. Watchers and Effects

Use watch() for reactive side effects and watchEffect() for automatic dependency tracking. 

import { watch, watchEffect } from 'vue'

const count = ref(0)
const message = ref('Hello')

// Watch specific reactive data
watch(count, (newValue, oldValue) => {
  console.log(`Count changed from ${oldValue} to ${newValue}`)
})

// Watch multiple dependencies
watch([count, message], ([newCount, newMessage]) => {
  console.log(`Count: ${newCount}, Message: ${newMessage}`)
})

// Automatic dependency tracking
watchEffect(() => {
  console.log(`Count is: ${count.value}`)
})

Migration from Options API

When migrating from Options API to Composition API, consider these strategies:

  • Start Small: Migrate one component at a time
  • Use Composables: Extract shared logic into composables
  • Gradual Migration: Mix both APIs during transition
  • Test Thoroughly: Ensure all functionality works correctly

Performance Considerations

The Composition API can help improve performance when used correctly:

  • Tree Shaking: Unused code is automatically removed
  • Bundle Splitting: Composables can be lazy-loaded
  • TypeScript Support: Better type checking and IDE support

Conclusion

The Vue.js 3 Composition API provides a powerful and flexible way to build reactive components. By following these best practices, you can write more maintainable, reusable, and performant Vue applications. Start experimenting with the Composition API in your next project and experience the benefits firsthand.
